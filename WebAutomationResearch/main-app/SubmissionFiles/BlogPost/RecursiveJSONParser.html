Recursive JSON Parsing

In this section, I’ll be talking about how the recursive JSON Parser works in the project. I’ll be going through step by step what each. But before you need to know about JSON Parsing and JSON Structure.

What is a JSON Parser

A JSON Parser means interpreting the data with the specific language that you are using at that moment. We’ll be using JavaScript in this case. JSON is usually read as a sting called the JSON string. This string follows the specification of the JSON. When we “parse” JSON, it means we are converting the string into a JSON object by following the JSON specification, where we can then use it in any way we wish.

JSON Structure

In this section, I’ll be going through the data structures that can be contained in a JSON. This is very important to allow you to further identify them in other JSONs and create “cases” for them that would manipulate them correctly. For example, if the String is a Boolean, create a checkbox instead of a text field.

Objects

Objects are surrounded with “{“ and “}”. Between them, several string values pairs can reside. For example, the object above contains, firstname, lastname, age, email.

String Value Pair

A string value pair is kinda of like variable declaration in some languages. “age” is known as the “String” or “Key” while “40” is known as the “value”.

A value can be many data types: String, number, object, array, boolean or null. You can declare multiple String Value pair, inside an object.

Array

Arrays are surrounded with “[“ and “]”. Between them, you can have multiple String Value Pairs. But you can even have array nested within an object.

Above you can see the array, contains 2 objects and each contains 2 String Value Pairs. This is important, as the project may need to return different React components based on if there’s an array or not!
