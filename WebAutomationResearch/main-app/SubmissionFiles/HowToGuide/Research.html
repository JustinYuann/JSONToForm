Research

What is a MVC CRUD Web Generator.

What Research Did I Do?

Before starting a project, I encourage you to research what your project is about instead of going straight into development. This means researching what the project does and how we could start to develop it. I researched parts like available frameworks, backends and database. To me it was very important to research more into what the project is to avoid being overwhelmed or intmidated by task. Below I'll be talking about what the project is about and ways you could do research.

What is MVC

MVC stands for Model View Controller, this is a software design pattern also known as a paradigm. For web development, a Model would be the Database that you're using such as MongoDB. The Controller would be the backend that you're using like Flask or Node.js and finally the View would be the frontend like React or Angular. For my project, I adapted MERN which states the combination of technologies to create a Web Application. It stands for MongoDB, Express, React and Node.js.


What is the Project's Purpose

The project's purpose is to automate the creation of generic web applications that interact with a database. The problem that was indentified is many software development companies are approached by companies to develop said application. My project would work by declaring variables of the web application in a JSON file such as types of users, permissions etc. The program would the generate skeletal code that the company or user can run or alter if need be. I tried to picture it like websites like SquareSpace or Wix where they create a website for you but more in-depth and more functionality like connecting your own database. Online Information

The internet is a great place to start after your lecturer's information of course! But just be careful on which one you choose but I'll get into that in the "Errors and Bugs" section of this page. Places that I learned from was mainly Youtube videos and some articles. I suggest finding videos on the core princples of the project you're planning to do, for example, it was finding out what was MVC and what was its purpose. I suggest Youtube in most cases unless there's no videos on it, Youtube to me is more engaging and easier to digest.

Online Tutorials

As advised by my lecturer, I was told to start to start bootcamp. A bootcamp is like a course that takes you from start to finish on anything you could think of. For me it was making a full stack application. However, before you jump into one, start to declare what languages you want to use, for example you want to use only Javascript so you’d choose React and MongoDB. For databases, they can vary to the application, for instance for an IM application you’d use something like Firebase. The places I looked for tutorials where Youtube and some tutorial sites like Skillshare and Udemy. But be careful that these tutorials aren’t too old, technology changes fast. I also suggest if you have free time just put a full stack tutorial and just watch it, you don't have to code with it but it was a great way for me to start learning some concepts of web development.
